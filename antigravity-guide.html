Vibe Coding 功能應用流程

🎯 什麼是 Vibe Coding？

Vibe Coding 是一種新的開發模式：你描述「你要什麼」，AI 寫程式。你扮演產品經理，AI 扮演工程師。重點是把你的「感覺」轉化為具體的產品。

標準工作流程

💭 1. 想清楚
描述你要做什麼
→
📝 2. Prompt
用自然語言告訴 AI
→
⚡ 3. AI 生成
AI 產生程式碼
→
👀 4. 檢視
看結果對不對
→
🔁 5. 迭代
修改直到滿意
①

定義產品需求（PRD Prompt）

在 Chat 視窗描述你要做什麼：「我要做一個個人網站，有自我介紹、作品集和聯絡表單。樣式要簡潔現代，用深色主題。」

prompt 範例
幫我建立一個個人作品集網站
- 首頁有大標題和簡短介紹
- 作品集區域用卡片排列
- 有聯絡我的表單
- 深色主題，現代感
請先建立 HTML 結構
②

讓 AI 生成基礎架構

AI 會產生 HTML / CSS / JS 程式碼。複製到對應的檔案，在瀏覽器打開看效果。

③

迭代調整細節

「把標題改成紅色」、「卡片之間加更多間距」、「把按鈕圓角改大一點」— 一步一步告訴 AI 你想改什麼。

④

加入功能

「幫我在作品集加上點擊彈出詳細介紹的功能」、「表單送出後顯示感謝訊息」。AI 會自動寫 JavaScript 邏輯。

⑤

修 Bug

有問題時直接截圖錯誤或把錯誤訊息貼給 AI：「這個錯誤是什麼意思？怎麼修？」

⚠️

Vibe Coding 的注意事項

AI 生成的程式碼不一定 100% 正確。要養成每次修改後都測試的習慣，不要一次叫 AI 做太多事。

安裝 Claude Code for VS Code

🤖 Claude Code 是什麼？

Claude Code 是 Anthropic（ChatGPT 競爭對手）推出的 AI 編程工具。相比 Antigravity，Claude Code 更擅長理解複雜的程式結構和大型專案，常被認為程式理解能力更強。

✅ Antigravity（Gemini）

  • 免費方案較慷慨
  • 與 Google 生態整合好
  • 適合初學者
  • 即時補全快速

⚡ Claude Code

  • 理解複雜程式碼強
  • 大型專案分析佳
  • 說明清晰易懂
  • 適合進階使用

1

在 Antigravity（Firebase Studio）安裝 Claude Code 擴充套件

或透過 npm 全域安裝

取得 API Key

🔗 為什麼需要 Git？

Git 是連結你的電腦和 GitHub 的橋梁。所有「把程式碼推上 GitHub」、「從 GitHub 下載最新版本」的操作，都需要先在電腦安裝 Git。它是一個免費的版本控制工具，安裝一次就好，之後在 Antigravity 和 Terminal 裡都能直接使用。

Windows 安裝

下載 .exe 安裝檔後執行，所有選項保持預設，一路點 Next 即可。安裝完成後，桌面右鍵選單會多出「Git Bash Here」。

驗證安裝（Git Bash 或 Command Prompt）
git --version
# 顯示類似：git version 2.44.0.windows.1
3

macOS 安裝

macOS 有三種方式，最推薦用 Homebrew：

方法 A：Homebrew（推薦）
# 先安裝 Homebrew（若尚未安裝）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 再安裝 Git
brew install git
方法 B：Xcode Command Line Tools（最簡單）

# 跳出安裝視窗後點「安裝」，完成後 Git 就有了

4

設定你的 Git 身份（必做）

安裝完後，告訴 Git 你是誰。這個資訊會出現在每次 Commit 記錄裡。

Terminal / Git Bash
# 設定名稱（填你的真名或暱稱）
git config --global user.name "你的名字"

# 設定 email（填你的 GitHub 帳號 email）
git config --global user.email "your@email.com"

# 確認設定
git config --list
5

在 Antigravity 確認 Git 整合

安裝 Git 後，重新開啟 Antigravity，左側欄的 🌿 Source Control 面板就會啟用。點擊它可以看到所有檔案修改、提交版本、推送到 GitHub，全部不用離開 IDE。

✅

安裝完就不用再管了

Git 是背景工具，安裝完之後你幾乎不會直接去碰它。日常操作都在 Antigravity 的 Source Control 面板或 Terminal 裡用幾個簡單指令完成。

GitHub 入門介紹

🐙 GitHub 是什麼？

GitHub 是全球最大的程式碼存放平台。你可以把它想成「程式碼的 Google Drive」— 存放、備份、版本控制、與他人協作都在這裡。免費帳號就能建立無限個公開/私人專案。

📦

Repository（倉庫）

一個專案就是一個 Repo。存放你的所有程式碼和歷史記錄。

🌿

Branch（分支）

建立分支來開發新功能，不影響主要程式碼。測試 OK 再合併。

📸

Commit（提交）

每次存檔就是一個 Commit，可以隨時回到任何一個歷史版本。

🔀

Pull Request

想把你的修改合併進主程式碼時，發一個 PR 讓別人 Review。

🌐

GitHub Pages

免費把你的網站部署到網路上！只要一個設定就能有自己的網址。

⭐

Star & Fork

Star 是書籤，Fork 是複製別人的專案到自己帳號繼續開發。

快速開始步驟

1

註冊 GitHub 帳號

前往 github.com，點擊 Sign up，填入 email、密碼、用戶名稱，完成信箱驗證。

2

建立第一個 Repository

登入後點右上角 + → New repository，填入名稱，勾選「Add a README」，點擊 Create。

3

安裝 Git

前往 git-scm.com 下載安裝 Git。這是連結 VS Code 和 GitHub 的橋梁。

terminal — 確認安裝
git --version
# 應顯示類似: git version 2.44.0
4

設定 Git 身份

terminal
git config --global user.name "你的名字"
git config --global user.email "你的email@example.com"

Antigravity 串接個人 GitHub

🔗 為什麼要串接？

串接後，你可以在 VS Code 裡直接：推送程式碼到 GitHub、讓 AI 幫你撰寫 Commit 訊息、自動建立 PR，全程不用離開編輯器。

1

在 VS Code 安裝 GitHub 相關擴充

在 Extensions 搜尋並安裝：GitHub Pull Requests 和 GitLens（選擇性但很有用）。

2

登入 GitHub 帳號

VS Code 左下角點擊帳號圖示 → Sign in to GitHub → 瀏覽器跳出後授權。或在 Source Control 面板點擊「Clone Repository」時登入。

3

讓 Antigravity 看到你的 GitHub Repo

在 Gemini Chat 視窗輸入 @，可以看到引用本機專案的選項。若想讓 AI 了解整個專案結構，用 @workspace 引用整個工作區。

4

用 AI 生成 Commit 訊息

改完程式後，在 Source Control 面板的訊息欄旁有 Gemini 小圖示，點擊後 AI 會自動產生描述你修改了什麼的 Commit 訊息。

AI 生成範例
# AI 自動產生的 Commit 訊息:
feat: add responsive navigation menu

- Add hamburger menu for mobile devices
- Implement smooth scroll to sections
- Fix header overlap on scroll
🚀

GitHub Pages 一鍵發布

把你的網站 push 到 GitHub 後，到 Repo Settings → Pages → 選擇 main branch → Save，幾分鐘內就有 username.github.io/repo-name 的公開網址！

把 GitHub 倉庫抓到本機

1

方法一：VS Code 直接 Clone

按 Ctrl+Shift+P → 輸入 Git: Clone → 貼上 GitHub Repo 網址 → 選擇存放位置。

方法二：Terminal 命令

terminal — 基本 Git 流程

核心觀念：為什麼要加自動化測試？

😱 Vibe Coding 的痛點

AI 改 A 功能時，不知不覺壞了 B 功能。等你發現的時候，已經不知道是哪次修改造成的。解法：建立自動化測試，每次 AI 改完就自動驗證所有功能是否正常。

①

建立 SPEC 文件（規格書）

請 AI 根據你的需求產生一份 SPEC.md，列出所有功能點和預期行為。這是測試的基礎。

prompt 範例
根據我們目前建立的購物車功能，
幫我生成一份 SPEC.md，列出：
1. 所有功能點
2. 每個功能的預期行為
3. 邊界條件（例如：數量為0、商品不存在）
②

用 AI 生成測試案例

把 SPEC.md 丟給 AI，請它產生對應的測試程式碼。

prompt 範例
根據 @SPEC.md 的內容，
幫我用 Jest 生成完整的測試檔案
涵蓋所有功能點，包含正常情況和邊界情況
③

設定 GitHub Actions 自動跑測試

每次 push 到 GitHub，自動執行所有測試。AI 改壞東西時立刻發現。

prompt 範例
幫我在 .github/workflows/ 建立一個
CI 工作流程，每次 push 時自動：
1. 安裝依賴
2. 執行 npm test
3. 測試失敗時發 email 通知
④

Antigravity Workflow：測試驅動開發

每次叫 AI 做新功能的標準流程：

📝 寫需求
→
🧪 AI 寫測試
→
⚡ AI 寫功能
→
✅ 跑測試
→
🐙 Push
⑤

讓 AI 維護測試

功能改變時，連測試也讓 AI 一起更新：「我修改了購物車的折扣邏輯，請同步更新測試案例。」

🏆

進階心法：把 AI 當新人工程師

好的 Vibe Coding 不是叫 AI 做所有事，而是你定義規則、AI 執行，你 review 結果。你是架構師，AI 是執行者。每次大的修改前先跑測試確認基線，改完再跑一次確認沒有破壞。
