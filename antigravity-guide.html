Vibe Coding 功能應用流程

🎯 什麼是 Vibe Coding？

Vibe Coding 是一種新的開發模式：你描述「你要什麼」，AI 寫程式。你扮演產品經理，AI 扮演工程師。重點是把你的「感覺」轉化為具體的產品。

標準工作流程

💭 1. 想清楚
描述你要做什麼
→
📝 2. Prompt
用自然語言告訴 AI
→
⚡ 3. AI 生成
AI 產生程式碼
→
👀 4. 檢視
看結果對不對
→
🔁 5. 迭代
修改直到滿意
①

定義產品需求（PRD Prompt）

在 Chat 視窗描述你要做什麼：「我要做一個個人網站，有自我介紹、作品集和聯絡表單。樣式要簡潔現代，用深色主題。」

prompt 範例
幫我建立一個個人作品集網站
- 首頁有大標題和簡短介紹
- 作品集區域用卡片排列
- 有聯絡我的表單
- 深色主題，現代感
請先建立 HTML 結構
②

讓 AI 生成基礎架構

AI 會產生 HTML / CSS / JS 程式碼。複製到對應的檔案，在瀏覽器打開看效果。

③

迭代調整細節

「把標題改成紅色」、「卡片之間加更多間距」、「把按鈕圓角改大一點」— 一步一步告訴 AI 你想改什麼。

④

加入功能

「幫我在作品集加上點擊彈出詳細介紹的功能」、「表單送出後顯示感謝訊息」。AI 會自動寫 JavaScript 邏輯。

⑤

修 Bug

有問題時直接截圖錯誤或把錯誤訊息貼給 AI：「這個錯誤是什麼意思？怎麼修？」

⚠️

Vibe Coding 的注意事項

AI 生成的程式碼不一定 100% 正確。要養成每次修改後都測試的習慣，不要一次叫 AI 做太多事。

🔗 為什麼需要 Git？

Git 是連結你的電腦和 GitHub 的橋梁。所有「把程式碼推上 GitHub」、「從 GitHub 下載最新版本」的操作，都需要先在電腦安裝 Git。它是一個免費的版本控制工具，安裝一次就好，之後在 Antigravity 和 Terminal 裡都能直接使用。

git-scm.com — 官方下載頁
Git

前往 Git 官網下載

打開瀏覽器，前往 git-scm.com，網站會自動偵測你的作業系統並顯示對應的下載按鈕。

Windows 安裝

下載 .exe 安裝檔後執行，所有選項保持預設，一路點 Next 即可。安裝完成後，桌面右鍵選單會多出「Git Bash Here」。

驗證安裝（Git Bash 或 Command Prompt）
git --version
# 顯示類似：git version 2.44.0.windows.1
3

macOS 安裝

macOS 有三種方式，最推薦用 Homebrew：

方法 A：Homebrew（推薦）
# 先安裝 Homebrew（若尚未安裝）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 再安裝 Git

設定你的 Git 身份（必做）

在 Antigravity 確認 Git 整合

✅

安裝完就不用再管了

Git 是背景工具，安裝完之後你幾乎不會直接去碰它。日常操作都在 Antigravity 的 Source Control 面板或 Terminal 裡用幾個簡單指令完成。

GitHub 入門介紹

🐙 GitHub 是什麼？

GitHub 是全球最大的程式碼存放平台。你可以把它想成「程式碼的 Google Drive」— 存放、備份、版本控制、與他人協作都在這裡。免費帳號就能建立無限個公開/私人專案。

📦

Repository（倉庫）

一個專案就是一個 Repo。存放你的所有程式碼和歷史記錄。

🌿

Branch（分支）

建立分支來開發新功能，不影響主要程式碼。測試 OK 再合併。

📸

Commit（提交）

每次存檔就是一個 Commit，可以隨時回到任何一個歷史版本。

🔀

Pull Request

想把你的修改合併進主程式碼時，發一個 PR 讓別人 Review。

🌐

GitHub Pages

免費把你的網站部署到網路上！只要一個設定就能有自己的網址。

⭐

Star & Fork

Star 是書籤，Fork 是複製別人的專案到自己帳號繼續開發。

快速開始步驟

1

註冊 GitHub 帳號

前往 github.com，點擊 Sign up，填入 email、密碼、用戶名稱，完成信箱驗證。

2

建立第一個 Repository

登入後點右上角 + → New repository，填入名稱，勾選「Add a README」，點擊 Create。

3

安裝 Git

前往 git-scm.com 下載安裝 Git。這是連結 VS Code 和 GitHub 的橋梁。

terminal — 確認安裝
git --version
# 應顯示類似: git version 2.44.0
4

設定 Git 身份

terminal
git config --global user.name "你的名字"
git config --global user.email "你的email@example.com"

Antigravity 串接個人 GitHub

🔗 為什麼要串接？

串接後，你可以在 VS Code 裡直接：推送程式碼到 GitHub、讓 AI 幫你撰寫 Commit 訊息、自動建立 PR，全程不用離開編輯器。

在 VS Code 安裝 GitHub 相關擴充

2

登入 GitHub 帳號

3

讓 Antigravity 看到你的 GitHub Repo

在 Gemini Chat 視窗輸入 @，可以看到引用本機專案的選項。若想讓 AI 了解整個專案結構，用 @workspace 引用整個工作區。

4

用 AI 生成 Commit 訊息

改完程式後，在 Source Control 面板的訊息欄旁有 Gemini 小圖示，點擊後 AI 會自動產生描述你修改了什麼的 Commit 訊息。

把 GitHub 倉庫抓到本機

1

方法一：VS Code 直接 Clone

按 Ctrl+Shift+P → 輸入 Git: Clone → 貼上 GitHub Repo 網址 → 選擇存放位置。

2

方法二：Terminal 命令

# 進入資料夾

3

修改後推回 GitHub

terminal — 基本 Git 流程
# 查看有哪些檔案被修改
git status

這是最接近真實工作場景的開發流程：從設計稿出發，讓 AI 幫你理解設計意圖、寫出規格書（PRD），再逐步把功能做出來。全程用自然語言跟 AI 對話，不需要自己寫任何指令。

視覺稿/截圖
→
📋 AI 產生 PRD
功能規格書
→
🧱 切版/架構
HTML+CSS
→
⚙️ 功能開發
JS 邏輯

🧪 測試驗收
確認完成

階段一：Figma 設計稿 → AI 理解設計

1

匯出 Figma 截圖

在 Figma 選取你要實作的頁面或元件，按 Ctrl+Shift+E 匯出成 PNG 圖片。或直接截圖也可以。

✅

完成指標

手上有一張清楚的設計稿截圖，可以看到頁面佈局、文字、顏色和元件。

2

把圖片拖進 Antigravity Chat

打開 Gemini Chat 視窗（左側 ✦ 圖示），直接把設計稿截圖拖放進對話框，然後用自然語言說：

你對 AI 說的話
這是我的設計稿截圖。
請幫我分析這個頁面的：
1. 整體佈局結構
2. 有哪些主要元件（按鈕、卡片、導覽列等）
3. 顏色主題和字型風格
4. 互動功能（點擊、hover 效果等）
✅

完成指標

AI 能說出頁面有哪些元件、什麼顏色、什麼佈局，而且描述跟你的設計稿吻合。

階段二：AI 產生 PRD（功能規格書）

3

請 AI 寫 PRD

基於剛才的設計分析，請 AI 產生一份完整的功能規格書：

你對 AI 說的話
根據剛才的設計稿分析，
幫我寫一份 PRD.md 功能規格書，包含：
- 頁面目的和使用者目標
- 所有功能點的詳細說明
- 每個元件的行為規則（例如：按鈕點擊後做什麼）
- 邊界情況處理（例如：空白狀態、載入中、錯誤）
- 預期的驗收標準
✅

完成指標

專案根目錄出現 PRD.md，裡面有清楚的功能列表和驗收標準，讀完後你知道每個功能應該怎麼運作。

4

審查並調整 PRD

仔細讀 AI 產生的 PRD，如果有不符合你預期的地方，直接告訴 AI 修改：

你對 AI 說的話（舉例）
PRD 的第三點不對，按鈕點擊後不是跳新頁面，
是在同一頁面展開一個下拉選單。請修正。

另外請加上：用戶未登入時，
點擊任何功能都要先導向登入頁面。
✅

完成指標

你對 PRD.md 的每一行都滿意，這份文件代表你對這個功能的完整期待。

階段三：逐步開發功能

5

先做 HTML 結構和 CSS 樣式

告訴 AI 先做靜態畫面，不要急著加 JavaScript 邏輯：

你對 AI 說的話
根據 @PRD.md 的規格，
先幫我把 HTML 結構和 CSS 樣式做出來，
要跟設計稿一樣的顏色、字型和間距。
暫時不用加任何 JavaScript 互動功能。
完成後告訴我預覽畫面對不對。
✅

完成指標

在瀏覽器打開 index.html，畫面佈局和設計稿至少 80% 相符。

6

一次只做一個功能

從 PRD 選第一個功能，單獨請 AI 實作：

你對 AI 說的話
根據 @PRD.md 的「功能三：下拉選單」，
幫我實作這個功能。
要求：
- 點擊按鈕展開選單
- 點選任一項目後選單收起
- 選中的項目顯示 checkmark
實作完後提醒我測試哪些情況。
✅

完成指標

按照 AI 的測試清單一一確認，全部通過才算這個功能完成，再進行下一個。

7

遇到問題直接說給 AI 聽

不用懂任何程式語法，直接描述你看到的問題：

你對 AI 說的話
我點下拉選單的第二個選項後，
選單沒有收起來，還是展開的狀態。
而且選中的 checkmark 出現在第一項不是第二項。
請幫我找出原因並修正。
✅

完成指標

你描述的問題 AI 都能修好，不需要自己看程式碼。

8

全部功能完成後做最終驗收

所有功能做完後，請 AI 幫你對照 PRD 做驗收：

你對 AI 說的話
請根據 @PRD.md 的「預期驗收標準」，
列出所有我應該測試的情境清單，
包含正常情況和邊界情況，
我會逐一手動測試確認。
✅

完成指標

跑完 AI 給的所有測試情境，全部通過 ✓。功能開發完成，可以進入部署階段。

【Antigravity Workflow 實戰】市場不會為爛產品買單！建立自動化測試，解決 Vibe Coding 痛點

核心觀念：為什麼要加自動化測試？

😱 Vibe Coding 的痛點

AI 改 A 功能時，不知不覺壞了 B 功能。等你發現的時候，已經不知道是哪次修改造成的。解法：建立自動化測試，每次 AI 改完就自動驗證所有功能是否正常。

①

建立 SPEC 文件（規格書）

請 AI 根據你的需求產生一份 SPEC.md，列出所有功能點和預期行為。這是測試的基礎。

prompt 範例
根據我們目前建立的購物車功能，
幫我生成一份 SPEC.md，列出：
1. 所有功能點
2. 每個功能的預期行為
3. 邊界條件（例如：數量為0、商品不存在）
②

用 AI 生成測試案例

把 SPEC.md 丟給 AI，請它產生對應的測試程式碼。

prompt 範例
根據 @SPEC.md 的內容，
幫我用 Jest 生成完整的測試檔案
涵蓋所有功能點，包含正常情況和邊界情況
③

設定 GitHub Actions 自動跑測試

每次 push 到 GitHub，自動執行所有測試。AI 改壞東西時立刻發現。

prompt 範例
幫我在 .github/workflows/ 建立一個
CI 工作流程，每次 push 時自動：
1. 安裝依賴
2. 執行 npm test
3. 測試失敗時發 email 通知
④

Antigravity Workflow：測試驅動開發

每次叫 AI 做新功能的標準流程：

📝 寫需求
→
🧪 AI 寫測試
→
⚡ AI 寫功能
→
✅ 跑測試
→
🐙 Push
⑤

讓 AI 維護測試

功能改變時，連測試也讓 AI 一起更新：「我修改了購物車的折扣邏輯，請同步更新測試案例。」

🏆

進階心法：把 AI 當新人工程師

好的 Vibe Coding 不是叫 AI 做所有事，而是你定義規則、AI 執行，你 review 結果。你是架構師，AI 是執行者。每次大的修改前先跑測試確認基線，改完再跑一次確認沒有破壞。

10

GCP 開 VM，把服務部署上線 進階

☁️ 為什麼要部署到 GCP？

在自己電腦上跑的網站，只有你自己看得到。部署到 Google Cloud Platform（GCP）的虛擬機器（VM）後，任何人在世界上任何地方，用瀏覽器就能打開你的網站。全程告訴 AI 你要做什麼，AI 幫你生成所有指令，你只需要複製貼上確認。

☁️ 建立 GCP 帳號
→
🖥️ 開一台 VM
→
🔗 SSH 連進去
→
📦 部署程式碼
→
🌐 對外公開

階段一：建立 GCP 帳號與專案

1

建立 GCP 帳號

前往 cloud.google.com，用 Google 帳號登入。新帳號有 $300 美元免費額度（90天），開 VM 測試完全夠用。需要輸入信用卡，但不會自動扣款，額度用完才會詢問你是否升級。

✅

完成指標

成功登入 GCP Console，右上角看到你的 Google 帳號，首頁顯示「剩餘試用額度 $300」。

2

請 AI 幫你建立 GCP 專案

回到 Antigravity，告訴 AI 你要做什麼：

你對 AI 說的話
我要在 GCP 建立一個新專案來部署我的網站。
請告訴我：
1. 在 GCP Console 哪裡建立新專案
2. 專案名稱要怎麼取（有什麼限制）
3. 建立後我應該看到什麼畫面確認成功
✅

完成指標

GCP Console 左上角的專案選單裡看到你剛建立的專案名稱，且已被選取。

階段二：開一台虛擬機器（VM）

3

請 AI 推薦 VM 規格

告訴 AI 你的網站用途，讓他推薦最省錢的配置：

你對 AI 說的話
我要在 GCP 開一台 VM 來跑一個 Node.js 網頁應用，
預計同時使用人數不超過 10 人，這是練習用的。
請推薦最省錢的 VM 規格（機器類型、磁碟大小、作業系統），
並告訴我在 GCP Console 哪裡設定，
每個欄位填什麼值。
💡

AI 通常會推薦

e2-micro 或 e2-small、Ubuntu 22.04 LTS、10GB SSD。每月費用約 $5~10 美元，免費額度夠用幾個月。

✅

完成指標

VM 建立成功，在「Compute Engine → VM 執行個體」頁面看到你的 VM 狀態是綠色的「執行中」。

4

設定防火牆規則開放 HTTP/HTTPS

VM 預設不對外開放，需要告訴 GCP 允許網頁流量進來：

你對 AI 說的話
我的 GCP VM 建好了，但現在外部無法用瀏覽器連進來。
請告訴我怎麼在 GCP Console 設定防火牆規則，
允許 HTTP（port 80）和 HTTPS（port 443）的流量進來。
請說明每個設定欄位要填什麼。
✅

完成指標

「VPC 網路 → 防火牆規則」看到允許 port 80 和 443 的規則，且已套用到你的 VM。

階段三：連進 VM 並部署程式碼

5

用 SSH 連進 VM

在 GCP Console 的 VM 列表，點擊「SSH」按鈕，瀏覽器會彈出一個黑色視窗（就是 Terminal），這就表示你已經連進 VM 了。

🖥️ GCP SSH Terminal — VM 內部
你@你的VM名稱:~$
// 你已經在 VM 裡面了，可以開始操作
✅

完成指標

瀏覽器出現黑色 SSH 視窗，命令提示符顯示你的 VM 名稱，例如：你的名稱@my-vm:~$

6

請 AI 生成環境安裝指令

你不需要記任何指令，全部讓 AI 生成，你複製貼上就好：

你對 AI 說的話
我已經用 SSH 連進 GCP 的 Ubuntu 22.04 VM 了。
我的網站是用 Node.js + Express 寫的。
請幫我生成：
1. 更新系統套件的指令
2. 安裝 Node.js 和 npm 的指令
3. 安裝 Git 的指令
4. 安裝 PM2（讓服務在背景跑）的指令
每個指令執行後我應該看到什麼訊息確認成功？
✅

完成指標

在 VM 裡執行 node --version 和 npm --version 都能看到版本號碼。

7

從 GitHub 把程式碼拉到 VM

你對 AI 說的話
我的程式碼在 GitHub 上，
repo 網址是 https://github.com/我的帳號/我的專案
請幫我生成：
1. 把程式碼 clone 到 VM 的指令
2. 安裝 npm 依賴的指令
3. 確認能正常啟動的指令
✅

完成指標

執行啟動指令後，Terminal 顯示「Server running on port 3000」或類似成功訊息。

8

設定 PM2 讓服務永遠在線

你對 AI 說的話
我的 Node.js 服務可以跑了，
但關掉 SSH 視窗後服務就停了。
請幫我用 PM2 把服務設定成：
1. 在背景持續執行
2. VM 重啟後自動啟動
3. 怎麼查看服務狀態是否正常
✅

完成指標

執行 pm2 list 看到服務狀態是 online，關閉 SSH 視窗後過 1 分鐘再打開，服務依然在線。

階段四：對外公開，取得網址

9

用 VM 的外部 IP 直接訪問

GCP VM 建立後會有一個「外部 IP」，在 VM 列表可以看到。在瀏覽器輸入 http://你的外部IP:3000 就能看到你的網站。

⚠️

注意：IP 是暫時的

每次重啟 VM，外部 IP 可能會變動。要請 AI 教你設定「靜態外部 IP」讓網址固定。

✅

完成指標

在任何一台電腦或手機的瀏覽器輸入 IP，都能看到你的網站正常運作。

10

（選）設定自己的網域名稱

如果你有自己的網域（如 mysite.com），請 AI 指引你完成最後一步：

你對 AI 說的話
我的 GCP VM 外部 IP 是 34.xx.xx.xx
我在 GoDaddy 買了一個網域叫 mysite.com
請告訴我：
1. 怎麼在 GoDaddy 設定 DNS 指向這個 IP
2. 怎麼在 VM 設定 Nginx 讓 port 80 轉到我的服務
3. 怎麼申請免費的 HTTPS 憑證（Let's Encrypt）
✅

最終完成指標

在瀏覽器輸入 https://mysite.com，看到你的網站，網址列顯示 🔒 鎖頭（HTTPS），代表整個部署流程完成！
